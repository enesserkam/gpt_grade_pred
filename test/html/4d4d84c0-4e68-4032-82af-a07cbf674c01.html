Certainly! Let's proceed with tuning the hyperparameters max_depth and criterion.

Explanation of Hyperparameters:

  1. max_depth:

    • Definition: The maximum depth of the tree. It controls the maximum number of levels in the decision tree.
    • Usage: Controlling the depth of the tree is crucial to prevent overfitting. A deeper tree can capture more complex relationships in the training data but may lead to overfitting, especially with small datasets.
    • Tuning: By tuning this parameter, you can find the optimal depth that balances between capturing enough information and avoiding overfitting. GridSearchCV will search through different values for max_depth and determine the one that results in the best performance during cross-validation.

  2. criterion:

    • Definition: The function used to measure the quality of a split. It determines the criterion for making decisions in the tree.
    • Usage: The decision tree uses this criterion to evaluate different splits at each node. The two commonly used criteria are "gini" and "entropy."
    • Tuning: By testing both "gini" and "entropy" during hyperparameter tuning, you can find which criterion performs better for your specific dataset. It's a good practice to test both to see if one significantly outperforms the other.

Code for Hyperparameter Tuning:

python
from sklearn.model_selection import GridSearchCV

# Define the decision tree classifier
dt_classifier = DecisionTreeClassifier(random_state=42)

# Define the hyperparameters to tune
param_grid = {
    'max_depth': [None, 5, 10, 15, 20],  # Vary the maximum depth
    'criterion': ['gini', 'entropy']  # Test both gini and entropy criteria
}

# Create the GridSearchCV object
grid_search = GridSearchCV(estimator=dt_classifier, param_grid=param_grid, cv=5, scoring='accuracy')

# Fit the model to the training data
grid_search.fit(X_train, y_train)

# Print the best hyperparameter values
best_params = grid_search.best_params_

# Print the best validation accuracy

print("Best Validation Accuracy:", best_accuracy)

This code uses GridSearchCV to search through different values of max_depth and criterion, using 5-fold cross-validation to find the best hyperparameter values that result in the highest accuracy on the validation set.

Certainly! To calculate the information gain, we need to compute the entropy of the parent node and the average entropy of the child nodes. The entropy is calculated using the formula:

Entropy(S)=−p1⋅log⁡2(p1)−p2⋅log⁡2(p2)−…−pk⋅log⁡2(pk)Entropy(S) = - p_1 \cdot \log_2(p_1) - p_2 \cdot \log_2(p_2) - \ldots - p_k \cdot \log_2(p_k)Entropy(S)=−p1​⋅log2​(p1​)−p2​⋅log2​(p2​)−…−pk​⋅log2​(pk​)

where pip_ipi​ is the probability of class iii in the set SSS.

Let's calculate the information gain for the first split:

python
# Get the first split feature and threshold
first_split_feature = final_model.tree_.feature[0]
first_split_threshold = final_model.tree_.threshold[0]

# Filter data based on the first split
left_child_indices = X_train.iloc[:, first_split_feature] <= first_split_threshold
right_child_indices = ~left_child_indices

# Calculate the probabilities for the parent node
p_parent_healthy = y_train.value_counts(normalize=True)['healthy']
p_parent_overweight = y_train.value_counts(normalize=True)['overweight']
p_parent_underweight = y_train.value_counts(normalize=True)['underweight']

# Calculate the entropy of the parent node
entropy_parent = - p_parent_healthy * np.log2(p_parent_healthy) - \
                 p_parent_overweight * np.log2(p_parent_overweight) - \
                 p_parent_underweight * np.log2(p_parent_underweight)

# Calculate the probabilities for the left child node
p_left_healthy = y_train[left_child_indices].value_counts(normalize=True)['healthy']
p_left_overweight = y_train[left_child_indices].value_counts(normalize=True)['overweight']
p_left_underweight = y_train[left_child_indices].value_counts(normalize=True)['underweight']

# Calculate the entropy of the left child node
entropy_left = - p_left_healthy * np.log2(p_left_healthy) - \
               p_left_overweight * np.log2(p_left_overweight) - \
               p_left_underweight * np.log2(p_left_underweight)

# Calculate the probabilities for the right child node
p_right_healthy = y_train[right_child_indices].value_counts(normalize=True)['healthy']
p_right_overweight = y_train[right_child_indices].value_counts(normalize=True)['overweight']
p_right_underweight = y_train[right_child_indices].value_counts(normalize=True)['underweight']

# Calculate the entropy of the right child node
entropy_right = - p_right_healthy * np.log2(p_right_healthy) - \
                p_right_overweight * np.log2(p_right_overweight) - \
                p_right_underweight * np.log2(p_right_underweight)

# Calculate the average entropy of the children
average_entropy_children = (sum(left_child_indices) / len(y_train)) * entropy_left + \
                           (sum(right_child_indices) / len(y_train)) * entropy_right

# Calculate information gain
information_gain = entropy_parent - average_entropy_children

# Print the information gain
print(f"Information Gain on the first split: {information_gain}")
